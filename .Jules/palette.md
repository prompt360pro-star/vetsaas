## 2024-05-18 - Accordion Widget Accessibility
**Learning:** Custom disclosure widgets like the `Accordion` component lack native semantic relationships, requiring explicit `aria-expanded`, `aria-controls` on the trigger, `role="region"`, and `aria-labelledby` on the content wrapper for screen readers, as well as distinct `focus-visible` classes to ensure keyboard navigation visibility without disrupting mouse users. Additionally, decorative icons (like `ChevronDown`) need `aria-hidden="true"` to prevent unnecessary announcements.
**Action:** When creating or maintaining custom interactive UI components in `packages/web`, systematically verify that appropriate ARIA relationships and `focus-visible` utility classes are applied, ensuring compliance with accessible interactive widget patterns.
